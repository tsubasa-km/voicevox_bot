# ユーザー向け言い換えガイド

## 方針

- 技術用語より、利用者が受ける変化を先に書く。
- 「何が変わったか」と「どう便利になったか」を短く書く。
- 断定できない内容は書かない。

## 言い換え例

- 「話者ID解決ロジックを追加」
  -> 「話者を設定していないユーザーでも、毎回同じ声で読み上げられるようになりました。」

- 「未指定時の話者選択ルールを統一」
  -> 「通常の読み上げとAPI経由の読み上げで、声の選び方を統一しました。」

- 「ノーマル style のみに限定」
  -> 「自動選択される声を通常スタイルに限定し、不自然な読み上げが起きにくくなりました。」

- 「返却 settings.speakerId を解決後に変更」
  -> 「設定更新後に返る話者IDが、実際に使われる値になりました。」

- 「pitch/speed 更新時に speaker を強制保存しない」
  -> 「速度やピッチだけを変更しても、話者設定が意図せず変更されないようになりました。」
